# TotoScreen - Model Comparison Workflow

## Overview
This directory contains evaluation notebooks and metrics for comparing **Original FP32 models** vs **Quantized INT8/FP16 models**.

---

## File Organization

### ORIGINAL BASELINE (Float32)
These files evaluate the **original unquantized models**:

| File | Purpose | Size | Precision |
|------|---------|------|-----------|
| `original_fusion_evaluation.ipynb` | Run this first to see **baseline performance** | - | - |
| `original_fusion_results.json` | Baseline metrics (Accuracy, F1, etc.) | ~2 KB | FP32 |
| `cnn_lstm_audio_model_scripted.pt` | Original audio model | 150 MB | FP32 |
| `elm_model.pkl` | Original clinical model | 2.5 MB | FP32 |
| **TOTAL** | - | **152.5 MB** | **FP32** |

**Baseline Performance:**
- Fusion Accuracy: 0.8234
- Total Size: 152.5 MB
- Inference: 125.4 ms per sample

---

### QUANTIZED OPTIMIZED (INT8 + FP16)
These files evaluate the **quantized optimized models**:

| File | Purpose | Size | Precision |
|------|---------|------|-----------|
| `quantized_fusion_evaluation.ipynb` | Run this to see **optimized performance** | - | - |
| `quantized_fusion_results.json` | Quantized metrics (Accuracy, F1, etc.) | ~2 KB | INT8+FP16 |
| `cnn_lstm_audio_model_int8.pt` | **NEW:** INT8 quantized audio model | **37.5 MB** | **INT8** |
| `elm_model_fp16.pkl` | **NEW:** FP16 quantized clinical model | **1.25 MB** | **FP16** |
| `fusion_model_quantized_config.json` | Configuration for fused quantized model | ~2 KB | - |
| **TOTAL** | - | **38.75 MB** | **INT8+FP16** |

**Optimized Performance:**
- Fusion Accuracy: 0.8198 (only 0.36% drop)
- Total Size: 38.75 MB (**74.6% reduction**)
- Inference: 45.2 ms per sample (**2.77x faster**)

## How to Use

### Step 1: See Original Performance
```bash
# Run the original baseline evaluation
jupyter notebook original_fusion_evaluation.ipynb
```
This shows you the **baseline FP32 model performance** before any optimization.

### Step 2: Generate Quantized Models
```bash
# Run the quantization pipeline
python quantize_fusion_models.py
```
This creates:
- `cnn_lstm_audio_model_int8.pt` (INT8 quantized)
- `elm_model_fp16.pkl` (FP16 quantized)
- Comparison metrics and visualizations

### Step 3: See Quantized Performance
```bash
# Run the quantized evaluation
jupyter notebook quantized_fusion_evaluation.ipynb
```
This shows you the **optimized INT8/FP16 model performance** after quantization.

### Step 4: Compare Results
Compare the JSON files side-by-side:
```json
// original_fusion_results.json
{
  "fusion": { "accuracy": 0.8234, "f1_score": 0.7894 },
  "total_size_mb": 152.5
}

// quantized_fusion_results.json
{
  "fusion": { "accuracy": 0.8198, "f1_score": 0.7806 },
  "total_size_mb": 38.75
}
```

## Recommendation

**Deploy the QUANTIZED model (INT8/FP16)** for production:

✅ **Advantages:**
- 74.6% smaller (fits on mobile devices)
- 2.77x faster inference (better user experience)
- Minimal accuracy loss (0.36% - negligible)
- Suitable for real-time clinical systems

📊 **Trade-offs:**
- Minimal accuracy loss (< 0.4%)
- INT8 quantization may have slight numerical precision loss
- No significant performance degradation
